# 일일 복권 (Lotto Gacha)

FF14 "미니 갹토팟" 스타일의 경품 추첨 프론트엔드 MVP.
3×3 칸에 1~9가 랜덤 배치되고, 처음 1칸이 공개된 채 시작합니다.
플레이어는 최대 3칸을 더 열어본 뒤 직선 한 줄(가로3·세로3·대각선2 = 8줄)을 고르고,
그 줄 세 숫자의 합계(6~24)로 상품이 정해집니다.

## 실행

```bash
npm install
npm run dev      # 개발 서버
npm run build    # 프로덕션 빌드
```

## 게임 흐름

1. **복권 선택** — 뒷면 카드가 원형(coverflow) 캐러셀로 늘어서고, 좌우 화살표로 넘기며 하나 선택
2. **숫자 공개** — 처음 1칸 자동 공개 → 가려진 칸을 최대 3칸까지 열람
3. **라인 선택** — 그리드 주변 화살표로 8개 라인 중 하나 선택
4. **결과** — 합계 → 상품 확정, 재고 1 차감

## 설정 (`src/config/`)

- **`tickets.ts`** — 복권 종류. 복권마다 상품(S~D 등급 각 1종)·초기 재고·뒷면 색/이모지를 지정합니다.
  `buildPrizeSet({ S:{name, initialStock}, ... })` 로 상품과 보상표를 한 번에 만듭니다.
- **`grades.ts`** — 등급별 표시 색상 테마.

### 합계 → 등급 규칙

중앙값 15에서 멀수록 희귀합니다. `|합계 - 15|` 기준:

| 거리 | 합계 | 등급 |
|---|---|---|
| 9 | 6, 24 | S |
| 7~8 | 7·8·22·23 | A |
| 5~6 | 9·10·20·21 | B |
| 3~4 | 11·12·18·19 | C |
| 0~2 | 13~17 | D |

15는 항상 마방진(4-9-2/3-5-7/8-1-6)으로 만들 수 있어, 재고가 최저 등급(D)만 남아도
보드 생성이 보장됩니다.

## 재고 보장 로직 (`src/domain/board.ts`)

보드를 생성할 때 **8개 라인이 모두 재고 있는 상품으로 매핑되는 숫자 배치**를
reject-sampling으로 찾습니다. 따라서 플레이어가 어떤 줄을 고르든 **품절 상품은 절대 당첨되지
않습니다.** 재고가 소진돼 유효 보드를 못 만드는 복권은 선택 화면에서 SOLD OUT으로 비활성화됩니다.

- 재고는 `localStorage`(`lotto-gacha` 키)에 저장돼 새로고침 후에도 유지됩니다.
- 헤더의 **재고 초기화** 버튼으로 초기값으로 되돌릴 수 있습니다(운영용).

## 상품 이미지

`public/images/prizes/<ticketId>/<grade>.png` 에 파일을 넣고 `tickets.ts`의 각 상품 `imageUrl`을
지정하면 결과 카드에 표시됩니다. 없으면 🎁 플레이스홀더로 렌더됩니다.
